Using components

What is a component?

Components are reusable elements of the user interface. Such as buttons, text inputs and checkboxes.

You can use individual components in different patterns and contexts. For example, you can use the text input component to ask for an email address, an NHS number or someone's name.

How to use components

The prototype kit has two ways to use components. You can either use HTML or a Nunjucks macro. (We will look at the pros of using Nunjucks vs HTML later on)

To use a component, you simply copy the code snippet from the relevant page from the design system, and paste the snippet into the relevant file and block within said file (usually inside the content block).

For example, if I wanted to add an inset text component to this page I would do the following:

  1. Browse to the inset text page (opens in a new tab) on the design system website
  2. Select whether I wish to use HTML or Nunjucks, and copy the code snippet
  3. Paste the code snippet inside the relevant block inside the file for the page you wish the component to be displayed on and edit it accordingly
  4. ave the file and refresh the browser to see the component on your page like so:

What is Nunjucks?

Nunjucks is a templating language that's powered by JavaScript. It's a way of extending HTML that lets us add logic, loops and various other capabilities.

Nunjucks allows our templates to be dynamic, even though at the end, they'll generate static HTML. It's sort of like Sass, but for HTML rather than CSS.

Nunjucks vs HTML

  • Nunjucks allows you to easily pass data across pages - for example you could enter your name into a text input on one page, then on a check your answers page easily show the value you entered previously.
  • You can also change the HTML that's generated by passing options into the Nunjucks code - for example I could have a component which is simply a square box, which could have an option for colour, and depending on the value I assign, could quickly and easily change the whole component to that colour.
  • Nunjucks allows you to use loops, for example I could have 4 people, and I want to use a card component that shows their name and job title, in HTML I would have to write out 4 components and change the information manually, Nunjucks could allow me to use a foreach loop and automatically replace the name and job title accodingly.
  • Arguably the best feature, if you use Nunjucks, your application will be easier to update when a new version of the toolkit is released. Instead of changing every page that uses the component, the macro will be updated and automatically update all the pages using it.
