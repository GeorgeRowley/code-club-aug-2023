Using custom layouts

This section of the tutorial is more of a 'bonus' section. Up to now we have covered using GOV.UK components in an 'NHS page' and/or customising it.

But what do you do if for example you want your prototype to have a journey where it leaves an 'NHS page' and directs to a 'GOV.UK page' - maybe in the scenario where you want to take a payment?

Creating a custom layout

When you have been making pages you may have noticed that your page is 'extending' from a file called layout.html which in turn extends a file called template.html. These files are basically where the header, footer and footer live and references to your stylesheet etc.

You can in theory use the 'blocks' to overwrite these sections and apply a GOV.UK header instead, but I personally find creating both a new layout and template file is the easier solution, below are the steps to do so, again using GOV.UK as the example.

Important: Please note

For this to work, you must have followed the steps at the top of the previous page, and this method only allows you to use HTML components not Nunjucks. Further steps required for using Nunjucks.

  1. Inside docs/views create a file, template-gov.html and inside app/views create a file called layout-gov.html
  2. Copy the contents of this file template-gov.html (opens in a new tab) to your own template-gov.html file
  3. Copy the contents of this file layout-gov.html (opens in a new tab) to your own layout-gov.html file
  4. Create your page as you would normally, inside app/views and use the following code as your 'boilerplate':

    <!-- Extends the layout from /views/layout.html -->
    {% extends 'layout-gov.html' %}

    <!-- Set the page title -->
    {% block pageTitle %}
    Page title here
    {%endblock %}

    <!-- For adding page content -->
    {% block content %}
    {%endblock %}
  5. Fill out your page with components from GOV.UK design system as necessary
